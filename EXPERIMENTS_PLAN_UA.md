# План програми експериментів (UA)

Дата: 2026-03-08  
Поточний профіль середовища: `esp32s3_csi_v1`

## 1) Призначення

Цей план задає єдині правила для чотирьох типів експериментів:

1. `distance`
2. `interference_v1`
3. `angle`
4. `static_sign_v1`

## 2) Спільні правила для всіх експериментів

- Фіксувати TX/RX геометрію в межах однієї сесії.
- Писати зрозумілий `exp_id`/`dataset_id` з датою.
- Зберігати умови кімнати в `room_id`, `notes`, `scenario_tags`.
- Перед кожною сесією запускати:
  - `./tools/exp --list-devices`
  - `./tools/exp --list-target-profiles`
  - dry-run для перевірки CSI-потоку

## 3) Мінімальний обсяг даних (рекомендація)

Distance:

- 4-6 дистанцій
- 2-3 повтори на дистанцію
- 200-500 пакетів на repeat

interference_v1:

- 9 сценаріїв у наборі `core`
- 3 runs на сценарій
- 1500 пакетів на run

Angle:

- 8 кутів (`0..315` крок 45)
- 2 runs
- 150-300 пакетів на trial

static_sign_v1:

- 5 runs для `baseline`
- 5 runs для `hands_up`
- 20s на run (або еквівалент у пакетах)

## 4) Порядок виконання у лабораторний день

1. Підняти TX (`run_tx_laptop.sh`).
2. Перевірити RX і serial доступ.
3. Виконати `distance`.
4. Виконати `interference_v1`.
5. Виконати `angle`.
6. Виконати `static_sign_v1`.
7. Запустити аналіз і перевірити артефакти в `out/`.

## 5) Контроль якості

- Відхилити run, якщо `records=0` або є явні пропуски.
- Додати примітку про аномалії в `notes`.
- Не змішувати різні умови кімнати під одним `exp_id`.
