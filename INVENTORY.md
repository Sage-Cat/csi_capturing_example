## CSI Capture Path (ESP32-C3/C3-compatible serial stream)

### Serial source and framing

- Serial line source: `csi_capture.capture.serial_lines()`
- Parser: `csi_capture.parser.parse_csi_line()`
- Required marker: line must contain `CSI_DATA,`
- Parsing logic:
  - CSV parse of line
  - fields[1] => `esp_timestamp`
  - fields[2] => `mac`
  - fields[3] => `rssi`
  - last field => CSI payload list string parsed with `ast.literal_eval`
- Accepted row schema from parser:
  - `timestamp` (host ms)
  - `rssi`
  - `csi` (interleaved I/Q int list)
  - `esp_timestamp`
  - `mac`

### Serial device access checks

- `csi_capture.capture.ensure_serial_port_access(port)` validates existence and RW access.
- Existing permission guidance references `dialout` group.
- Existing default in config-driven flow is `/dev/esp32_csi`.

## Current Dataset Layout and Metadata

### Legacy distance layout (preserved in scripts)

- `experiments/<exp_id>/meta.json`
- `experiments/<exp_id>/<scenario>/run_<run_id>/distance_<X>m.jsonl`
- Per-run `manifest.json` generated by `scripts/run_rx_laptop.sh`.

### Unified capture layout (`csi_capture.experiment`)

- `experiments/<exp_id>/<experiment_type>/run_<run_id>/manifest.json`
- `experiments/<exp_id>/<experiment_type>/run_<run_id>/trial_<trial_id>/capture.jsonl`

### Existing metadata fields found in capture rows/manifests

- Common packet fields: `timestamp`, `rssi`, `csi`, `esp_timestamp`, `mac`
- Experiment fields: `exp_id`, `experiment_type`, `run_id`, `trial_id`, `repeat_index`
- Context fields: `scenario` and/or `scenario_tags`, `device_path`, `room_id`, `environment_notes`
- Ground truth fields: `distance_m` (distance), `angle_deg` (angle)

## Current Training / Feature / Evaluation Pipeline

- Distance modeling is implemented in script form:
  - `tools/analyze_wifi_distance_measurement.py`
- Feature extraction currently in that script:
  - CSI amplitude derivation from interleaved I/Q
  - per-packet stats (`mean_amp`, `std_amp`, `median_amp`, `topK_amp_mean`)
  - optional downsampled amplitude vectors + PCA components
- Models currently in that script:
  - RSSI log-distance model
  - CSI kNN regressor (sklearn) with ridge fallback
- Evaluation/reporting currently in that script:
  - grouped train/test split by run
  - MAE/RMSE/etc tables
  - figures and markdown report in `out/distance_measurement`
